Aperçu et publication d'une expérience Web

Dans cette rubrique, vous apprendrez comment prévisualiser et publier une expérience Web.

Les expériences Web peuvent facilement être prévisualisées pendant le développement. Les liens permettant de prévisualiser l'expérience peuvent également être partagés. Une fois l'expérience terminée, elle peut être publiée pour la rendre accessible au public.

Aperçu d'une expérience Web

L'aperçu d'une expérience Web vous permet de voir comment l'expérience va s'afficher sur les appareils mobiles, tablettes et ordinateurs de bureau. Pour prévisualiser une expérience Web, modifiez l'expérience, puis cliquez sur Aperçu dans l'éditeur d'expérience.

Deux options sont disponibles :

  • Aperçu réactif - Ouvre l'expérience dans une nouvelle fenêtre de navigateur pour un aperçu
  • Get Shareable Link - Fournit une URL de prévisualisation afin que l'aperçu puisse être partagé avec d'autres personnes.

L'URL partageable générée est valide pendant 72 heures.

Remarque : Si vous avez configuré un délai d'expiration de session dans le cadre de n'importe quel paramètre de contrôle d'accès, le délai d'expiration de 72 heures remplacera le délai d'expiration de votre session configuré. Lorsque l'expérience est publiée, le délai d'expiration de la session (le cas échéant) sera appliqué.

Publication d'une expérience Web

La publication d'une expérience Web la rend accessible aux utilisateurs et génère une URL statique pour le site. Pour publier une expérience, procédez comme suit :

Remarque : Si vous avez récemment apporté des modifications aux métadonnées vidéo ou aux listes de lecture, ces modifications peuvent prendre quelques minutes avant d'apparaître dans les expériences publiées.
  1. Cliquez sur le nom de l'expérience pour ouvrir le site dans l'Éditeur d'expérience.
  2. Cliquez sur Publish.
  3. Dans la boîte de dialogue de confirmation de publication, cliquez sur Publier maintenant.
  4. L'expérience sera publiée et l'URL du site sera affichée.

Le bcvp0rtal domaine prend en charge SSL par défaut, de sorte que les sites peuvent également être consultés à l'aide de https ://, aucun certificat SSL ou configuration n'est requis.

Une fois l'expérience publiée, l'URL sera affichée dans le cadre des détails du site sur le Projets page d'accueil.

Publication des modifications apportées à une expérience Web

Lorsque vous apportez des modifications à une expérience, elle doit être republiée. Les expériences qui doivent être republiées afficheront un statut de l'expérience des modifications non publiées.

Pour republier les modifications, suivez les étapes décrites dans le Publication d'une expérience Web section de ce sujet.

Remarque : Il se peut que vous deviez vider le cache du navigateur et les cookies si les modifications n'apparaissent pas.

Mise hors ligne d'une expérience Web

En prenant une expérience hors ligne, elle n'est plus disponible à son URL. Pour mettre une expérience hors ligne, cliquez sur une ligne, puis cliquez sur Prendre hors ligne. Pour confirmer que l'expérience doit être mise hors ligne, cliquez sur Déconnexion. Pour que l'expérience soit à nouveau disponible, l'expérience doit être republiée. Une fois l'expérience hors ligne, elle ne sera plus accessible via son URL.

Remarque : Le fait de mettre l'expérience hors ligne ne supprime pas l'expérience.
