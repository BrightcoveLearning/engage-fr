Aperçu et publication d'une expérience d'application mobile

Dans cette rubrique, vous apprendrez comment prévisualiser et publier une expérience d'application mobile.

L'aperçu d'une expérience d'application mobile vous permet de voir comment l'expérience va s'afficher sur un appareil mobile. Pour prévisualiser une expérience d'application mobile, modifiez l'expérience, puis cliquez sur Aperçu dans l'éditeur d'expérience.

Deux options sont disponibles :

  • Aperçu réactif - Ouvre l'expérience dans une nouvelle fenêtre de navigateur pour un aperçu
  • Get Shareable Link - Fournit une URL de prévisualisation afin que l'aperçu puisse être partagé avec d'autres personnes.

L'URL partageable générée est valide pendant 72 heures.

Remarque : Si vous avez configuré un délai d'expiration de session dans le cadre de n'importe quel paramètre de contrôle d'accès, le délai d'expiration de 72 heures remplacera le délai d'expiration de votre session configuré. Lorsque l'expérience est publiée, le délai d'expiration de la session (le cas échéant) sera appliqué.

Publication d'une expérience d'application mobile

La publication d'une expérience d'application mobile génère une clé d'accès. La clé d'accès est requise par Brightcove Engage™ app (iOS et Android) pour accéder à l'expérience. Pour plus d'informations sur l'installation et l'utilisation de l'application Brightcove Engage, consultez Utilisation de l'application Brightcove Engage.

Pour publier une expérience d'application mobile, procédez comme suit :

Remarque : Si vous avez récemment apporté des modifications aux métadonnées vidéo ou aux listes de lecture, ces modifications peuvent prendre quelques minutes avant d'apparaître dans les expériences publiées.
  1. Cliquez sur le nom de l'expérience pour l'ouvrir dans l'Éditeur d'expérience.
  2. Cliquez sur Publish.
  3. Dans la boîte de dialogue de confirmation de publication, cliquez sur Publier maintenant.
  4. L'expérience sera publiée et la clé d'accès sera affichée.

    Une fois l'expérience publiée, la clé d'accès s'affiche dans les détails de l'expérience sur la page d'accueil Expériences.

Utilisation de l'application Brightcove Engage

Après la publication d'une expérience d'application mobile, la clé d'accès est entrée dans l'application mobile Brightcove Engage pour afficher l'expérience. Pour plus d'informations sur l'installation et l'utilisation de l'application Brightcove Engage, consultez Utilisation de l'application Brightcove Engage.

Publication des modifications apportées à une expérience

Lorsque vous apportez des modifications à une expérience, elle doit être republiée. Les expériences qui doivent être republiées afficheront un statut de l'expérience des modifications non publiées.

Pour republier les modifications, suivez les étapes décrites dans le Publication d'une expérience d'application mobile section de ce sujet.

Mise hors ligne d'une expérience d'application mobile

Si une expérience d'application mobile est mise hors ligne, elle ne sera plus accessible à partir d'applications mobiles natives. Pour mettre une expérience d'application mobile hors ligne, cliquez sur une expérience d'application mobile, puis sur Mettre hors ligne. Pour confirmer que l'expérience doit être mise hors ligne, cliquez sur Déconnexion. Pour que l'expérience soit à nouveau disponible, l'expérience doit être republiée.

Remarque : Le fait de mettre l'expérience hors ligne ne supprime pas l'expérience.
