Configuration des domaines personnalisés et SSL

Dans cette rubrique, vous apprendrez comment configurer des domaines personnalisés et SSL pour une expérience Web.

Lorsque vous créez une expérience Web pour la première fois, un domaine aléatoire lui est attribué au format <site-slug>.bcvp0rtal.com . Par exemple, http://site-148393.bcvp0rtal.com. Lorsque l'expérience est publiée, les internautes pourront accéder au site dans ce domaine. Brightcove Engage permet de publier des expériences Web dans un domaine fourni par Brightcove ou dans votre propre domaine personnalisé.

Les expériences Web peuvent également être attribuées à plusieurs URL permettant aux éditeurs d'utiliser le même site pour différentes fonctions sans avoir à dupliquer le site. Par exemple, si vous avez un site vidéo que vous souhaitez utiliser dans une campagne et comme portail vidéo principal, vous pouvez marquer différentes URL et les affecter toutes au même site. Il n'est pas nécessaire de dupliquer le site et vous pouvez supprimer les URL en toute simplicité dès qu'elles ne sont plus nécessaires.

Utilisation d'un domaine fourni par Brightcove

Les domaines fournis par Brightcove vous permettent de créer rapidement et facilement un domaine pour votre expérience Web sans avoir à travailler avec votre service informatique pour créer un CNAME personnalisé ou enregistrer un nouveau nom de domaine auprès d'un bureau d'enregistrement. Ils sont disponibles « hors de la boîte » et peuvent être mis en œuvre en un clic de bouton. Les domaines disponibles sont :

  • brightcoveengage.video
  • bcvp0rtal.com (qui est attribué à Web Experiences par défaut)

Tous prennent en charge l'accès SSL. Pour accéder au site par une connexion SSL, il vous suffit d'ajouter https à l'URL. Par exemple, https://mysite.brightcoveengage.video

Utilisation d'un domaine personnalisé (enregistré par l'éditeur)

Web Experiences peuvent également être hébergées sur votre propre domaine personnalisé que vous enregistrez. Cela vous permet de nommer l'URL de façon cohérente avec la dénomination de votre site internet ou de votre URL actuelle. Pour ce faire, vous devrez d'abord enregistrer le nom de domaine puis configurer un enregistrement CNAME personnalisé auprès de votre hébergeur. Pour obtenir des informations sur la configuration d'un enregistrement CNAME, consultez la section ci-dessous.

Création d'un domaine personnalisé

Pour créer un domaine personnalisé, procédez comme suit :

  1. Ouvrez Brightcove Engage.
  2. Ouvrez le Projets et cliquez sur le Réglages lien en haut de la page
  3. Dans la navigation de gauche, cliquez sur le Domaines personnalisés et SSL lien.
  4. Cliquez sur Create Custom Domain.
  5. Sélectionnez une option de domaine à utiliser :
    • brightcoveengage.video - Le domaine du site aura le format <your domain>.brightcoveengage.video.
    • Personnalisé : sélectionnez cette option pour héberger le site sur votre propre domaine personnalisé. Lorsque vous sélectionnez cette option, vous devez également configurer un enregistrement CNAME auprès de votre hébergeur.
  6. Saisissez le nom de domaine. Un message s'affiche, indiquant si le domaine est valide ou non. Si le nom de domaine n'est pas valide, entrez un nouveau nom de domaine.
  7. Si vous publiez le site à l'aide d'un domaine fourni par Brightcove, il existe une option pour appliquer SSL sur ce domaine. Vérifier Appliquer SSL sur ce domaine pour activer une redirection depuis la version non SSL du site si un spectateur y accède.
  8. Si vous publiez le site sur votre propre domaine personnalisé, vous avez la possibilité de sécuriser le site via SSL. Cochez la case Use SSL pour sécuriser le site à via SSL. Pour plus d'informations sur l'utilisation de SSL, consultez Sécuriser une expérience Web avec SSL.
  9. Vérifier Activer la redirection pour rediriger cette URL vers une autre URL. Généralement, cela ne sera vérifié qu'après que ce domaine a été utilisé pendant un certain temps. Lorsque cette option est sélectionnée, entrez l'URL de redirection vers laquelle cette URL existante doit être redirigée. Cochez Utiliser une redirection permanente (code d'état HTTP 301) pour indiquer que la redirection est permanente. Lorsqu'une araignée de moteur de recherche trouve un code d'état 301 dans l'en-tête de réponse d'une page Web, elle comprend que cette page Web n'existe plus et remplace l'URL indexée par la nouvelle. Cette option conservera le trafic et le classement de votre ancienne page Web.
  10. Cliquez sur Save. Une liste des domaines configurés s'affiche.
    Remarque : Lorsque vous utilisez un domaine fourni par Brightcove, l'état SSL est toujours fourni. Cela indique qu'en ajoutant https:// au début de l'URL du site, la transmission des données entre votre navigateur et le site Brightcove Engage sera sécurisée.
  11. Vous pouvez créer des domaines supplémentaires selon vos besoins.

Pour modifier ou supprimer un domaine, cliquez sur le bouton Modifier () ou supprimer ().

Une fois les domaines personnalisés créés, ils peuvent être affectés à une expérience dans le cadre des propriétés d'expérience. Pour plus d'informations, consultez Attribution d'un domaine personnalisé à une expérience Web.

Configuration des enregistrements CNAME auprès de votre hébergeur

Remarque : les enregistrements CNAME peuvent être configurés que lorsque les domaines personnalisés ont été affectés à un site. IL N'EST PAS nécessaire de créer des enregistrements CNAME pour les sites utilisant des domaines fournis par Brightcove.

Les étapes de configuration d'un enregistrement CNAME (nom canonique) varient selon l'hébergeur. Vous devez configurer un enregistrement CNAME pour chaque domaine personnalisé que vous utiliserez. Si votre site est configuré pour utiliser SSL, l'enregistrement CNAME pointe vers une URL CloudFront. Voir Sécuriser une expérience Web avec SSL pour information. Notez que les domaines personnalisés doivent inclure un sous-domaine pour être configurés en tant que CNAME ex: engage doit être placé avant brightcovelearning.com afin que je puisse configurer mon CNAME.

La valeur CNAME s'affiche dans le cadre des paramètres Domaines personnalisés et SSL.

Dans cet exemple, pour le nom de domaine engage.brightcovelearning.com, il convient d'ajouter un enregistrement CNAME et un hôte de engage et pointez-le vers 5e84f4c7d58950766328a37d.bcvp0rtal.com.

Vous devrez peut-être contacter votre administrateur réseau ou votre fournisseur d'hébergement Web pour obtenir de l'aide sur cette étape. Il faut parfois patienter 48 heures avant que le changement de DNS ne soit fonctionnel.

Sécurisation des sites via SSL

Remarque : Lorsque vous travaillez avec des certificats SSL, ne partagez en aucun cas votre clé privée ni ne l'envoyez au support Brightcove.

Il existe deux options pour sécuriser une expérience de portail avec SSL :

  1. Pour les sites publiés à l'aide du slug de site attribué (c.-à-d. <site-slug>.bcvp0rtal.com ) ou utilisez un domaine fourni par Brightcove (<site>.brightcoveengage.video), aucune configuration n'est nécessaire pour prendre en charge https. Ajouter https: // au début de l'URL sécurisera la transmission des données entre votre navigateur et l'expérience. Vous pouvez également sélectionner Appliquer SSL sur ce domaine (recommandé) pour activer une redirection à partir de la version non-SSL du site si une visionneuse y va.
  2. Pour les sites hébergés sur un domaine personnalisé, vérifiez la Utiliser SSL option lors de la saisie de votre nom de domaine. Vous serez invité à saisir les informations de votre certificat. Pour plus d'informations, consultez Sécuriser une expérience Web avec SSL.
